# Product

## Register

product

## Users

Web3 developers, airdrop participants, and technical users verifying Merkle-based token claims. Primary context: desktop/laptop browser with MetaMask or similar wallet extension. Secondary context: mobile browsers for verification-only use.

## Product Purpose

A verified airdrop claim portal for Merkel Airdrop — a Merkle Tree-based Ethereum token distribution system. Users connect wallets, verify eligibility via Merkle proofs, and claim tokens on-chain. The interface serves both claim execution and educational exploration (understanding how Merkle proofs work).

## Brand Personality

Precision, trust, cryptographic elegance. Three words: **auditable**, **minimal**, **confident**. The interface should feel like a professional trading terminal crossed with a scientific instrument — not a generic SaaS dashboard.

## Anti-references

- Generic SaaS "AI-demo" gradient cards with 32px+ border-radius
- Tiny uppercase eyebrow labels on every section
- Numbered step markers (01/02/03) as default scaffolding
- Warm cream/beige backgrounds (the saturated AI default)
- Glassmorphism blur as decorative default
- Hero-metric templates with big numbers + gradient accents
- Side-stripe border accents
- Gradient text (`background-clip: text`)

## Design Principles

1. **Cryptographic minimalism** — Every pixel should feel intentional. No decorative clutter. The interface is a precision instrument, not a billboard.
2. **Dark surface discipline** — The deep dark surface IS the brand. Warmth comes from the amber accent, not from tinted backgrounds. The darkness conveys trust and focus.
3. **Information hierarchy through structure** — Data density is the product value. Dense, well-scaffolded information beats sparse hero sections.
4. **Motion as feedback, not decoration** — Animations communicate state transitions (hover, loading, success). No decorative ambient motion.
5. **Typography as confidence** — Tight but not cramped letterforms, generous weight contrast, monospace for anything cryptographic.

## Accessibility & Inclusion

WCAG 2.1 AA minimum. All interactive elements keyboard-navigable. Reduced motion respected. Body text contrast ≥4.5:1, large text ≥3:1. No color-only state indicators.
